HTML

HTML stands for Hypertext Mark-up Language, it was invented by Tim Berners-Lee in 1991. Tim Berners-Lee is also the inventor of the world wide web (www) in 1989, developed on the lanuage which became the pinnicle of how web sites are developed today.

www Browsers use HTML's set of markup symbols or codes to interpret how the items which make up the website are visually displayed on the screen. These 'Codes' and 'markup text' are often refferd to by the developer in HTML as 'Tags'. These elements are often divided into a start tag and '/' end tag such as "< p >< /p >" for paragraphs and "< a >< /a >" for links (without spaces).

Formally Recommended by the W3C, HTML has expanded and adapted through the years to incorporate new functions and improve usability. There are many different versions of HTML which have progressively changed the way websites are navigated, viewed and behave.

HTML+

In 1993, Dave Raggett drafted the superset of HTML which allowed designs to encompass features such as tables, figures and forms. It was gradually rolled out to improve the earlier formatting of HTML. HTML+ also introduced navigation menus and dividers to break down Large Documents to help reduce latency.

HTML+ was heavily influenced by the international standard of document markup at the time, the Standard Generalized Markup Language (SGML). SGML, defined by ISO 8879, markup structure help define document classes by the SGML Document Type Definition (DTD).

HTML+, unlike SGML, does not attempt to define each possible user but instead defines support for a very wide range of display types. HTML+ aids in keeping browser software as simple - limiting the complexity that can be handled.

HTML+ favors the user defined settings of the browser to impact the formatting of the webpage. Browsers can avoid problems with page sizes and missing fonts. HTML+ conserves indispensable differences that are often lost by lower level procedural formats, making it easier to carry out processes like indexing and convertion into other document formats.

HTML 2-3

The versions of HTML which followed where simple revision or adaptions, including new tags to help format the document or enable new images, tables or equations to be formed and displayed on the website.

HTML 2 - Developed by Tim Burners-Lee in September 22nd 1995 which featured improvements and conformance to various requirements

HTML 3 - (The evolution of HTML+) a Backwards compatible version of HTML drafted in March 1995 introducing tables, complex Maths, complex text-wrapping and formatting, subclass elements, paragraphs and support for non-graphical clients.

Further upgrades were also awarded a new version of HTML such as 3.0, 4.0 and 5.

An quick overview of the HTML Version and Differences;

  • HTML v1 - Linked Documents
  • HTML v2 - Linked Documents with Stylised text and tables
  • HTML v3 - Linked Documents with Stylised text and tables with some Browser Specific Features
  • HTML v4 - Re-evaluated Linked Documents with Stylised text and tables with some Browser Specific Features whilst removing clutter
  • HTML v5 - Linked Documents with the ability to specify Document Structures and multimedia elements such as CSS/JS

HTML 4 and XML = XHTML

HTML 4, published in 1999, was the first version of HTML document building which introduced the use of Cascading Style Sheets (CSS) as part of the HTML standard. HTML 4 is reliant on css documents for formating and display of the website providing prestine control of any information the page is hosting. As browsers improved to host HTML4 websites

XHTML, published in 2000, is output from certain Extensible Markup Language (XML) files, which use standard ASCII text, and mirrors or extends HTML versions with strict syntax rules, requiring perfect syntax to even run the XHTML, but it also allows developers to create their own objects and tags for each page in development. Introduces a uniform appearance to developed pages acoos a range of different browsers.

HTML5

Development started for HTML's 5th Version of standards in 2007 with the official standard being released by W3C in October 2014. HTML5 become popular and dominant in webpage development by 2010, cooly taking over the XHTML standard and the old news of HTML 4. HTML5 introduce a slueth of new Tags and elements to help define document format and structure such as header, footer, audio, video, navigation tags, article, section etc. These tags opened up the possibility of styling elements using CSS within the element rather then having to selectively style the entire page.

HTML5 relies heaviliy on additional scripts such as CSS and JavaScript for interactions, styling and any animations on the webpage. Though this version is easily the most interactive and adaptable when the webpage is developed using the combination of this HTML version, CSS and JavaScript/JQuery.
